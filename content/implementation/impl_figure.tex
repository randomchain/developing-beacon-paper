    \caption{Significant data flows in the beacon. Input collectors hash inputs and sends them to the input proxy which forwards the hashed inputs to the input processor. This input processor adds the inputs to the current Merkle tree as they arrive. When a computation node is finished with a job, it signals its readiness (1) to the input processor, and is either given data (2) or is requested to wait a bit (to keep computation nodes spaced apart in time). Once data is obtained, the delay function (3) is applied and the output (4) is sent through the output proxy to various publishers.}\label{fig:impl_figure}
